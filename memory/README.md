# EVA Memory (Long-Term Storage)

**Directory**: `memory/`  
**Purpose**: Archival, semantic, and structural long-term storage (LTM).  
**Version**: v9.6.2

---

## 📋 Overview

The **Memory Domain** is the immutable backbone of the organism. It stores the accumulated "Experience" of EVA across all versions and sessions. While `consciousness/` is volatile RAM, `memory/` is the encrypted, schema-validated hard drive.

---

## 📂 Memory Sectors

### 1. `archival_memory/`

The raw, chronological logs of all past episodes (Turn History).

### 2. `core_memory/`

Distilled stable traits and frequent patterns (Tier 2 of the 8-8-8 Protocol).

### 3. `sphere_memory/`

The highest level of "Wisdom" and global bias (Tier 3).

### 4. `context_storage/` (Archived Contexts)

Past snapshots of the `context_container` for specific historic turns.

### 5. `vector_store/`

Embeddings used for semantic search and Case-Based Reasoning (CBR).

---

## 📐 Governance

- **Write Protection**: Only the **MSP (Memory & Soul Passport)** engine has direct write authority to this directory.
- **Validation**: All data must pass structural and semantic verification before persistence.
- **Access**: The LLM accesses this directory primarily via the **Agentic RAG Service**.
